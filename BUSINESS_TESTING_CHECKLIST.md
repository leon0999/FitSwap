# 🚀 FitSwap 프로 사업가 테스트 체크리스트

**목표**: 실제 사용자가 사용하기 전에 모든 핵심 기능과 수익화 경로를 검증

---

## 1. 🎯 핵심 비즈니스 기능 테스트 (Critical Path)

### 1.1 음식 인식 → 영양 분석 → 대체품 추천 (전체 플로우)

#### 테스트 케이스 1: 브랜드 음식 (빅맥)
- [ ] 사진 업로드 → "Big Mac" 인식 확인
- [ ] 칼로리 563, 단백질 25g, 지방 30g 표시 확인
- [ ] Health Score v2 표시 (24/100, Poor 등급)
- [ ] 대체품 3개 이상 표시 확인
- [ ] 대체품마다 Health Score v2 + 뱃지 표시
- [ ] **수익화**: "Order Now" 버튼 클릭 → Uber Eats 열림 ✅
- [ ] **수익화**: "Near Me" 버튼 → 지도 모달 + 4개 배달 서비스 버튼

**예상 시간**: 2분 이내 전체 플로우 완료

#### 테스트 케이스 2: 집밥 (파스타)
- [ ] "Pasta with tomatoes and basil" 인식
- [ ] 복합 음식 계산 (재료별 영양소 합산) 확인
- [ ] 칼로리 88, 단백질 1.6g 표시
- [ ] Health Score v2: 100/100 (Excellent) 확인
- [ ] 품질 뱃지 표시 (Local, Clean Label 등)

#### 테스트 케이스 3: 오가닉 음식 (Organic Salad)
- [ ] "Organic" 키워드 감지 확인
- [ ] Health Score v2에서 Quality Score 50점 반영
- [ ] 🌿 USDA Organic 뱃지 표시
- [ ] 대체품도 오가닉 우선 추천 확인

## 2. 💰 수익화 경로 테스트 (Revenue Critical)

### 2.1 배달 딥링크 작동 확인

#### Uber Eats 링크
- [ ] "Order Now" 버튼 → 새 탭에서 Uber Eats 열림
- [ ] URL에 음식명 포함 확인 (예: `q=Grilled+Chicken+Salad`)
- [ ] 위치 정보 포함 확인 (예: `pl=37.7749,-122.4194`)
- [ ] 검색 결과가 관련 음식 표시하는지 확인

#### MapModal 배달 서비스
- [ ] "Near Me" 클릭 → 모달 열림
- [ ] 4개 배달 서비스 버튼 표시 (Uber Eats, DoorDash, Grubhub, Instacart)
- [ ] 각 버튼 클릭 → 해당 서비스 사이트 열림
- [ ] 음식명 검색 결과 확인

#### 추가 서비스 아이콘
- [ ] Alternative 카드 하단 3개 아이콘 (🏍️ 🍔 🛒)
- [ ] 각 아이콘 클릭 → 해당 서비스 열림

**목표**: 클릭 한 번으로 배달 주문 가능한 상태

## 3. 📊 데이터 정확성 테스트

- [ ] 집밥 파스타: 50-60 (Good/Fair)

### 3.3 대체품 추천

- [ ] 단백질 더 높은 음식 우선

## 4. 🎨 사용자 경험 (UX) 테스트

### 4.1 첫 방문 사용자 (Cold Start)
- [ ] 랜딩 페이지 → 15초 이내에 사용 방법 이해
- [ ] "Upload Food Photo" 버튼 명확하게 보임
- [ ] 사진 업로드 → 3초 이내 인식 시작
- [ ] 로딩 상태 표시 (스피너, 진행바)

### 4.2 반복 사용자
- [ ] "Analyze Another Food" 버튼 쉽게 찾음
- [ ] 이전 결과 초기화 확인
- [ ] 빠른 재분석 (캐싱 효과)

### 4.3 오류 상황
- [ ] 잘못된 이미지 (음식 아님) → 에러 메시지 표시
- [ ] 인식 실패 → "다시 시도" 버튼
- [ ] 네트워크 오류 → 명확한 안내 메시지
- [ ] API 한도 초과 → 적절한 대체 메시지

